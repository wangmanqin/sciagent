# SciAgent 项目原理说明（面试准备用）

这份文档用纯中文解释整个项目的原理，不涉及任何代码语法。
读完这份文档后，你应该能回答导师可能问的所有问题。

---

## 一、这个项目是什么？

SciAgent 是一个「科学计算智能体」。

用户用自然语言说一句话（比如"帮我优化散热器"），它能自动完成：
分析问题 → 写代码 → 执行代码 → 如果报错就自动修复 → 画图 → 给出分析报告

**它不是一个普通的聊天机器人（chatbot）。普通聊天机器人只能回答文字。
SciAgent 能真正地执行计算——它会写 Python 代码，在你的电脑上跑，拿到真实的计算结果。**

---

## 二、核心原理：ReAct 循环

ReAct = Reasoning（推理）+ Acting（行动）

这是 2023 年由 Yao 等人提出的一个框架（论文：ReAct: Synergizing Reasoning and Acting in Language Models）。

传统的大模型只能"想"（输出文字），不能"做"（执行动作）。
ReAct 的核心思想是：让大模型交替进行「想」和「做」。

具体到 SciAgent 中，一次完整的运行是这样的：

```
第1轮：
  [想] 大模型分析用户的问题，决定怎么解决
  [做] 大模型输出一段 Python 代码，说"请执行这段代码"
  [看] 我们的程序真的执行了代码，把结果（或报错）返回给大模型

第2轮（如果需要）：
  [想] 大模型看到执行结果，判断是否正确
  [做] 如果代码报错了，大模型修改代码，再次请求执行
  [看] 我们的程序再次执行，返回新结果

第N轮：
  [想] 大模型认为任务完成了
  [做] 输出最终的文字分析报告，不再请求执行代码
  → 循环结束
```

面试时你可以这样说：

> "我的 Agent 基于 ReAct 框架，核心是一个循环：大模型先推理分析问题，
> 然后通过工具调用执行代码，观察执行结果后再决定下一步。
> 这个循环持续进行，直到大模型认为任务完成。"

## 三、关键技术：Function Calling（工具调用）

这是让大模型能"做事"的核心机制。

### 它是怎么工作的？

1. 我们在调 API 的时候，除了发送用户的问题，还告诉大模型：
   "你有一个工具叫 run_python_code，它能执行 Python 代码"
2. 大模型收到问题后，不只返回文字，还可能返回一个结构化的指令：
   "我要调用 run_python_code，参数是这段代码：..."
3. 注意：大模型自己并不执行代码。它只是"说"它想执行什么。
   真正执行代码的是我们的程序。
4. 我们的程序收到这个指令后，真的去执行这段代码，把输出结果返回给大模型。

### 类比

想象你有一个很聪明的助手，但他被锁在一个房间里，只能通过纸条和你交流：

- 你递进去一张纸条："帮我算散热器的最优尺寸"
- 他递出一张纸条："请帮我在计算器上按这些步骤：..."
- 你按完后，把结果写在纸条上递回去
- 他看到结果后，递出最终分析报告

大模型就是那个助手，Function Calling 就是那个纸条传递机制。

面试时你可以这样说：

> "Function Calling 是大模型 API 提供的一种机制。我们定义好工具的名称、
> 描述和参数格式，大模型在需要时会返回一个结构化的工具调用请求，
> 我们的程序负责实际执行并将结果反馈给大模型。"

## 四、关键技术：System Prompt（系统提示词）

System Prompt 就是给大模型的「岗位说明书」。

在 SciAgent 中，我们的 System Prompt 告诉大模型：

1. 你是一个科学计算专家，擅长多目标优化
2. 你要按照「分析 → 写代码 → 执行 → 调试 → 可视化 → 解释」的流程工作
3. 写代码时必须遵守一些规则（比如必须用 print 输出结果，必须保存图片）
4. 如果代码报错，分析原因后重试，最多重试 5 次
5. 画图时必须用英文标注（因为中文会显示乱码）

### 为什么 System Prompt 很重要？

改了 Prompt，大模型的行为就会改变：

- 如果你不说"必须用 print"，大模型可能写了代码但不输出结果，我们就拿不到数据
- 如果你不说"不要用 plt.show()"，大模型会调用显示窗口，但服务器上没有屏幕，就会报错
- 如果你不说"用英文画图"，中文标注会变成方块

这就是 Prompt Engineering（提示词工程）——通过精心设计指令来控制大模型的行为。

面试时你可以这样说：

> "System Prompt 的设计是整个 Agent 最关键的部分。我通过反复测试发现了
> 几个关键问题，比如大模型生成的代码不输出结果、画图时中文乱码等，
> 都是通过在 Prompt 中加入明确的约束来解决的。"

## 五、关键技术：Self-Correction（自我修正）

这不是什么高级算法，原理非常简单：

1. 大模型写的代码执行后报错了
2. 我们把错误信息原封不动地发回给大模型
3. 大模型看到错误信息后，分析原因，修改代码，再次请求执行
4. 重复直到成功或达到最大重试次数

这就是为什么 Agent 有时候需要多轮才能完成——第 1 轮代码可能有 bug，
第 2 轮大模型看到报错后修复了，第 3 轮才成功。

面试时你可以这样说：

> "Self-correction 的实现很直接：把执行错误信息作为新的上下文反馈给大模型，
> 大模型会自动分析错误原因并修正代码。在我的测试中，大约 X% 的问题需要
> 经过 self-correction 才能成功。（这个数据你测试后填）"

## 六、项目的 5 个文件分别做什么？

不需要看代码，只需要知道每个文件的职责：

### tools.py — 工具层

职责：接收一段代码字符串 → 保存为文件 → 执行 → 返回输出结果
类比：就像一个「代码执行器」，大模型说"帮我跑这段代码"，这个文件负责真的去跑

### prompts.py — 提示词

职责：存放 System Prompt，也就是给大模型的岗位说明书
类比：就像一个「员工手册」，规定了大模型应该怎么工作

### llm.py — 大模型接口层

职责：负责和 DeepSeek API 通信。发送消息，接收回复，解析结果
类比：就像一个「翻译官」，把我们程序的格式翻译成 API 能理解的格式，再把 API 的返回翻译回来

### agent.py — Agent 主循环（最核心）

职责：实现 ReAct 循环。不断调用大模型，如果大模型要用工具就执行工具，
把结果喂回去，直到大模型给出最终回答
类比：就像一个「项目经理」，协调大模型和工具之间的来回沟通

### app.py — 前端界面

职责：用 Streamlit 做了一个网页，展示对话、Agent 的思考过程、生成的图表
类比：就像一个「展示窗口」，让用户能看到 Agent 在做什么

## 七、导师可能问的问题和你的回答

### Q1："你为什么做这个项目？"

> 我在科研中大量使用大模型辅助论文写作和数据分析，意识到大模型不仅能生成文字，
> 还能通过工具调用来执行实际的计算任务。我想验证这个想法，就以我最熟悉的
> 微通道散热器多目标优化为场景，搭建了一个能自主完成科学计算的 Agent。

### Q2："ReAct 框架是什么？和 Chain-of-Thought 有什么区别？"

> Chain-of-Thought 只是让大模型"想得更仔细"，输出的还是文字。
> ReAct 在此基础上加入了"行动"——大模型不仅思考，还能调用外部工具执行操作，
> 然后根据执行结果继续思考。这形成了一个「思考→行动→观察」的循环。

### Q3："你遇到了什么困难？怎么解决的？"

> （根据你的测试记录来回答，比如：）
> 最主要的问题是大模型生成的代码不稳定。比如它经常忘记 import 必要的库，
> 或者在画图时使用中文导致乱码。我通过迭代优化 System Prompt 解决了大部分问题，
> 比如明确要求"必须包含所有 import"和"图表标注必须用英文"。

### Q4："Agent 的成功率怎么样？"

> （根据你的 TEST_LOG.md 数据回答：）
> 在我测试的 10 个问题中，X 个一次成功，Y 个需要 self-correction 后成功，
> Z 个失败。失败的主要原因是...

### Q5："这个项目和直接写代码有什么区别？Agent 的价值在哪？"

> 传统方式需要人手动写代码、调试、画图、分析，整个过程可能需要几个小时。
> Agent 把这个流程自动化了：用户只需用自然语言描述问题，Agent 自动完成
> 从建模到出结果的全过程。更重要的是，Agent 具有自我修正能力——
> 代码报错后能自动诊断并修复，不需要人工干预。

### Q6："你觉得这个方向还有什么可以改进的？"

> 1. 当前 Agent 只有一个工具（代码执行），可以扩展更多工具，
>    比如文献检索、数据库查询、调用专业仿真软件（如 OpenFOAM）
> 2. System Prompt 是静态的，可以探索让 Agent 根据任务类型动态调整策略
> 3. 目前没有记忆机制，每次对话都是独立的，可以加入长期记忆
> 4. 可以做更系统的评测，建立一个科学计算的 benchmark

## 八、你需要了解的几个术语

| 术语                       | 含义                                       |
| -------------------------- | ------------------------------------------ |
| Agent（智能体）            | 能自主决策和行动的 AI 系统，不只是问答     |
| ReAct                      | "推理+行动"框架，让大模型交替思考和执行    |
| Function Calling           | API 提供的机制，让大模型能调用外部工具     |
| Tool Use                   | 和 Function Calling 同义，不同厂商叫法不同 |
| System Prompt              | 给大模型的系统级指令，控制其行为           |
| Prompt Engineering         | 通过设计提示词来优化大模型的输出           |
| Self-Correction            | 大模型根据错误反馈自动修正的能力           |
| Pareto Front（帕累托前沿） | 多目标优化中的最优解集合（你本来就懂这个） |
| NSGA-II                    | 经典的多目标进化优化算法（你本来就懂这个） |
